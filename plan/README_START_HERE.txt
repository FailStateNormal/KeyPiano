================================================================================
  keypiano — 开发启动说明
  日期：2026-06-14
================================================================================

【项目是什么】
  一个从零重写的 Windows 虚拟钢琴软件，参考 FreePiano 的键盘映射格式，
  作为 GPL v3 开源项目发布到 GitHub，仓库名：keypiano

【为什么重写而不是改 FreePiano】
  FreePiano 原版是 GPL，可以参考源码，但我们要做一个更现代、
  代码质量更高、自己完全掌控的版本。

【协作方式】
  - Claude 是主力开发，你负责把控方向和审查代码
  - 你的 C++ 算法基础够用，工程架构由 Claude 处理
  - 优先代码质量，目标是上 GitHub 的开源项目

================================================================================
  技术选型（已确定，不再讨论）
================================================================================

  UI          ：Qt6 Widgets（Windows）
  SF2 音色    ：FluidSynth 2.3+
  音频输出    ：RtAudio（WASAPI 后端，目标延迟 < 20ms）
  VST2 插件   ：VST2 SDK GPL v3（Steinberg 2022 年以 GPL v3 发布，无授权问题）
  构建系统    ：CMake 3.25 + vcpkg（manifest mode）
  单元测试    ：GoogleTest
  编译器      ：MSVC 2022（x64，唯一目标编译器）
  C++ 标准    ：C++20（2026-06-14 由 17 升级；MSVC 完整支持，便于后续开发）
  开源协议    ：GPL v3

================================================================================
  四个开发阶段
================================================================================

  Phase 1【核心引擎 - 能响】
    目标：按键能发出钢琴声，延迟 < 20ms，无 GUI
    交付：音频引擎、SF2 合成器、键盘钩子、.map 解析、.cfg 解析、
          CMake 工程骨架、vcpkg 依赖、基础单元测试
    验收：加载 FreePiano 的 freepiano.cfg + SF2 文件，按键有声音

  Phase 2【录制回放】
    目标：能录制演奏并精确回放
    交付：Recorder 模块、.kps 格式读写（我们自己的录制文件格式）
    验收：录制"小星星"保存为 .kps，加载后回放节奏一致

  Phase 3【Qt GUI】
    目标：完整可视化界面
    交付：主窗口、88 键钢琴可视化组件（QPainter 自绘，按键高亮动画）、
          音频反馈桥接、设置对话框、键位编辑器、SF2 选择器
    验收：按键时对应钢琴键高亮，松开后 0.1s 淡出

  Phase 4【VST2 + 发布】
    目标：支持 VSTi 插件，打包成可分发的单一 exe
    交付：VST2 宿主、VST 编辑器窗口嵌入 Qt、windeployqt 打包配置
    验收：加载 mdaPiano.dll 正常发声，打包后在无 Qt 开发环境的机器上运行

================================================================================
  .map 格式兼容说明（重要）
================================================================================

  FreePiano 有两套语法，我们都要解析：

  2.0 语法（flashpiano.map）：
    key a NoteOn 0 c2
    Sustain 0 Set 127
    keydown up KeySignature Inc 1
    label up Key+

  2.1 语法（freepiano.map）：
    Keydown A Note In_0 C3
    Keydown F5 Octave In_0 Inc 1
    Keydown ScrollLock Record
    Label Space FreePiano

  注意：2.0 中有个拼写错误 "KeyboardVeolcity"，解析器需要兼容这个错误写法。

================================================================================
  .kps 录制格式（我们自己设计的，不兼容 .fpm）
================================================================================

  为什么不用 .fpm：.fpm 是无文档的二进制格式（zlib 压缩），
  使用绝对时间戳且 GB2312 编码，格式设计差，不值得维护兼容。

  .kps 格式特点：UTF-8 文本，可读，可 Git 版本控制，微秒精度

  示例：
    # keypiano song file
    # Format version 1.0

    [meta]
    title     = 小星星
    created   = 2026-06-14T00:00:00Z
    duration  = 65340000
    keymap    = default.map
    soundfont = GeneralUser.sf2

    [events]
    # timestamp_us  type  channel  note  velocity
    0               CC    0        7     90
    965000          ON    0        72    127
    1465000         OF    0        72    0

================================================================================
  线程模型（三个线程）
================================================================================

  Hook Thread   ：Win32 WH_KEYBOARD_LL 全局键盘钩子，查表后写入无锁队列
  Audio Thread  ：RtAudio WASAPI 回调，消费队列事件，驱动 FluidSynth 渲染
  UI Thread     ：Qt 主线程，每 16ms 轮询反馈队列，更新钢琴键高亮

  音频线程铁律：禁止 malloc/new、禁止加锁等待、禁止 Qt 调用、禁止文件 I/O
